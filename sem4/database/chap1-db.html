Key Concepts

Data vs Information

  • Data: Raw facts (e.g., name, age, weight).
  • Information: Processed data with context (e.g., average student age).
  • Data is the foundation of knowledge; good decision-making relies on accurate information.

Database & DBMS

  • Database: Shared, integrated structure storing end-user data and metadata.
  • DBMS: Manages database structure and controls data access. Key functions:
    • Data storage/retrieval
    • Security management
    • Backup/recovery
    • Query processing (e.g., SQL)

  • Operational (day-to-day)
  • Analytical (data warehouses, OLAP)

File System Flaws

  • Structural Dependence: File access depends on structure.
  • Data Redundancy: Duplicate data causing inconsistency.
  • Data Anomalies:
    • Insertion: Forced duplication.
    • Deletion: Accidental data loss.
    • Modification: Inconsistent updates.

Poor vs Good Design

  • Poor Design: Redundant entries, anomalies (e.g., blank rows in employee skills table).
  • Good Design: Normalized tables (EMPLOYEE, SKILL, CERTIFIED) with relationships.

Summary

Databases enable efficient, secure data management. Proper design minimizes redundancy and anomalies. DBMS tools streamline storage, retrieval, and analysis, supporting decision-making across industries.
