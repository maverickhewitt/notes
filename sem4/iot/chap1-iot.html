Chapter 1: Things and Connections

1.1 What are Things?

1.1.1 The Internet of Things

  • IoT Presence: Embedded in daily life for quality-of-life improvements and industrial efficiency.
  • Cisco IoT System:
    • Addresses IoT complexity through six pillars:
      1. Network Connectivity
      2. Fog Computing
      3. Cybersecurity & Physical Security
      4. Data Analytics
      5. Management & Automation
      6. Application Enablement Platform

1.1.2 Building Blocks of an IoT System

ComponentRole
SensorsMeasure physical properties (e.g., temperature, motion)
ActuatorsConvert electrical signals to physical actions (e.g., motors, valves)
ControllersCollect sensor data, provide connectivity, and make decisions
IoT Process Flow: Sensors → Controllers → Actuators (via wired/wireless connections).

1.1.3 Processes in Controlled Systems

  • Feedback Loop: Output affects input to maintain desired state.
  • Control Systems:
    • Open-Loop: No feedback (e.g., simple timers).
    • Closed-Loop: Uses feedback (e.g., thermostat with PID controller).
PID Controllers: Use Proportional (P), Integral (I), and Derivative (D) calculations for precise control.

1.2 What are Connections?

1.2.1 Models of Communication

  • Layered Models:
    • OSI Model (7 layers)(PDNT-SPA)
    • TCP/IP Model (4 layers)
    • IoT World Forum Model (7 layers)
  • Standardization Bodies: Industrial Internet Consortium, OpenFog Consortium, Open Connectivity Foundation.

1.2.2 Layers of Connections

LayerProtocols/TechnologiesFunction
PhysicalCopper, Fiber, WirelessTransmit raw data bits
Data LinkEthernet, Wi-Fi (802.11), Bluetooth (802.15)Frame transmission & error detection
ApplicationMQTT, RESTDevice communication & data exchange
IoT Architectures: Device-to-Device, Device-to-Cloud, Device-to-Gateway-to-Cloud.

1.2.3 Privacy & Security Impacts

  • Metadata: "Data about data" (e.g., timestamps, device IDs).
  • Privacy Considerations:
    • Transparency in data collection
    • Limiting data access
  • Security Challenges:
    • Scale of IoT devices
    • Non-traditional device locations
    • Lack of firmware updates

1.3 Chapter Summary

  • IoT systems combine sensors, actuators, controllers, and software.
  • Closed-loop systems use feedback for precise control (e.g., PID controllers).
  • Communication models (OSI, TCP/IP) standardize IoT device interactions.
  • Security & privacy must be prioritized across all IoT layers.
