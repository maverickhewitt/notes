Chapter 2: Operating-System Structures

1. Operating System Services

  • User Interface (UI): CLI, GUI, or Batch
  • Program Execution: Load/run programs, handle errors
  • I/O Operations: Manage files and devices
  • File-System Manipulation: Create/delete files, manage permissions
  • Communications: Shared memory or message passing
  • Error Detection: Hardware/software error handling
  • Resource Allocation: CPU, memory, I/O devices
  • Accounting: Track resource usage
  • Protection & Security: Control access, user authentication
Key Diagram: OS services layer between hardware and user applications.

2. User OS Interfaces

Command-Line Interface (CLI)

  • Examples: Unix shells (Bash), Windows Command Prompt
  • Commands: ls, ping, iostat

Graphical User Interface (GUI)

  • Examples: Windows Explorer, macOS Aqua
  • Features: Icons, windows, touch/gesture support

Touchscreen Interfaces

  • Gestures, virtual keyboards, voice commands
  • Used in iOS, Android

4. System Programs & Services

  • File Management: cp, rm
  • Status Information: top, df
  • Programming Support: Compilers, debuggers
  • Background Services: Daemons (e.g., systemd)

7. System Boot & Debugging

Boot Process

  1. BIOS/UEFI initializes hardware
  2. Bootstrap loader (e.g., GRUB) loads kernel
  3. Kernel initializes hardware and mounts root filesystem

Debugging Tools

  • Logs: Crash dumps, syslog
  • Performance Tuning: top, Task Manager
  • Advanced Tools: DTrace (Solaris), strace
