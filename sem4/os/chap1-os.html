3. What is an Operating System?

  • An operating system (OS) is a program that acts as an intermediary between users and computer hardware.
  • Goals of an Operating System:
    • Execute user programs and make solving user problems easier.
    • Make the computer system convenient to use.
    • Use the computer hardware efficiently.

4. Computer System Structure

  • A computer system can be divided into four components:
    • Hardware: Provides basic computing resources (CPU, memory, I/O devices).
    • Operating System: Controls and coordinates the use of hardware among various applications and users.
    • Application Programs: Define how system resources are used to solve computing problems (e.g., word processors, compilers).
    • Users: People, machines, or other computers that interact with the system.

5. Operating System Definition

  • The OS serves as a resource allocator, managing all resources and deciding between conflicting requests for efficient and fair resource use.
  • It acts as a control program, controlling the execution of programs to prevent errors and improper use of the computer.
  • The kernel is the core component of the OS that runs at all times, while other components are either system programs or application programs.

6. Computer System Organization

  • Computer system operation involves:
    • One or more CPUs and device controllers connected through a common bus providing access to shared memory.
    • Concurrent execution of CPUs and devices competing for memory cycles.
    • Device drivers for each device controller to manage I/O operations.

7. Interrupt Handling

  • Interrupts signal events that require attention from the CPU, allowing the OS to respond to asynchronous events.
  • Types of interrupts:
    • Hardware Interrupts: Generated by hardware devices (e.g., I/O devices) to signal that they require processing.
    • Software Interrupts: Generated by programs (traps) due to errors (e.g., division by zero) or requests for OS services.
  • Interrupt handling involves:
    • Saving the state of the CPU and determining the appropriate interrupt service routine to execute.
    • Restoring the CPU state after the interrupt has been handled.

8. Process Management

  • A process is a program in execution, requiring resources such as CPU, memory, and I/O.
  • Key Concepts:
    • Process Lifecycle: Stages include New, Ready, Running, Waiting, and Terminated.
    • Single-threaded vs Multi-threaded: Single-threaded processes have one program counter, while multi-threaded processes have multiple counters for each thread.
    • Context Switching: The process of storing the state of a process so that it can be resumed later.
  • Activities include:
    • Creating and deleting both user and system processes.
    • Suspending and resuming processes based on resource availability.
    • Providing mechanisms for process synchronization (e.g., semaphores, mutexes).
    • Providing mechanisms for inter-process communication (IPC) such as message passing and shared memory.
    • Handling deadlocks through prevention, avoidance, detection, and recovery strategies.

9. Memory Management

  • Memory management determines what is in memory to optimize CPU utilization and response time.
  • Activities include:
    • Keeping track of which parts of memory are currently being used and by whom.
    • Deciding which processes (or parts thereof) and data to move into and out of memory.
    • Allocating and deallocating memory space as needed.

10. Storage Management

  • The OS provides a uniform, logical view of information storage, abstracting physical properties to logical storage units (files).
  • Activities include:
    • Creating and deleting files and directories.
    • Providing primitives to manipulate files and directories.
    • Mapping files onto secondary storage.
    • Backing up files onto stable (non-volatile) storage media.

11. Protection and Security

  • Protection mechanisms control access of processes or users to resources defined by the OS.
  • Security involves defending the system against internal and external attacks, including:
    • Denial-of-service attacks
    • Worms and viruses
    • Identity theft
    • Theft of service
  • Systems distinguish among users to determine who can do what, using user IDs and group IDs for access control.

12. Computing Environments

  • Types of computing environments include:
    • Traditional: Stand-alone general-purpose machines.
    • Mobile: Handheld devices like smartphones and tablets.
    • Distributed: Systems networked together, providing a unified interface.
    • Client-Server: Systems where clients request services from centralized servers.
    • Peer-to-Peer: Systems where all nodes act as both clients and servers.
    • Virtualization: Running multiple OSes on a single hardware platform.
    • Cloud Computing: Delivering computing resources as a service over the Internet.

13. Open-Source Operating Systems

  • Open-source operating systems are made available in source-code format, promoting user freedom and collaboration.
  • Examples include:
    • GNU/Linux
    • BSD UNIX (including core of Mac OS X)
  • Open-source software counters the copy protection and Digital Rights Management (DRM) movement.
